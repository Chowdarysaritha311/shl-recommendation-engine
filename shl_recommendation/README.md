# SHL Assessment Recommendation Engine

A RAG-based intelligent recommendation system that suggests relevant SHL assessments given a natural language query or job description.

---

## 🏗️ Architecture

```
User Query
    │
    ▼
[Query Understanding]   ← Gemini 1.5 Flash (extracts skills, job level, test types, duration)
    │
    ▼
[Semantic Search]       ← Gemini text-embedding-004 + Cosine Similarity
    │
    ▼
[Duration Filter]       ← Removes assessments exceeding stated time limit
    │
    ▼
[LLM Reranker]          ← Gemini reranks for relevance + balance (technical + behavioral)
    │
    ▼
[Top 5–10 Results]      ← Returned via API / shown in Streamlit UI
```

## 🚀 Setup (Step by Step)

### 1. Clone & Install

```bash
git clone https://github.com/YOUR_USERNAME/shl-recommendation-engine.git
cd shl-recommendation-engine
pip install -r requirements.txt
```

### 2. Set up environment variables

```bash
cp .env.example .env
# Edit .env and add your GEMINI_API_KEY
```

Get a free Gemini API key at: https://ai.google.dev/

### 3. Copy your dataset

```bash
# Copy the Gen_AI_Dataset.xlsx file into the data/ folder
cp /path/to/Gen_AI_Dataset.xlsx data/
```

### 4. Scrape SHL catalogue

```bash
python scripts/scraper.py
# This creates data/shl_assessments.csv with 377+ assessments
# Takes ~10-15 minutes
```

### 5. Run the API

```bash
uvicorn api:app --host 0.0.0.0 --port 8000 --reload
# API docs at: http://localhost:8000/docs
```

### 6. Run the frontend

```bash
# In a new terminal:
streamlit run app.py
# Opens at: http://localhost:8501
```

## ☁️ Deployment (Render.com — Free)

### Deploy the API:
1. Push code to GitHub
2. Go to https://render.com → New Web Service
3. Connect your repo
4. Build command: `pip install -r requirements.txt`
5. Start command: `uvicorn api:app --host 0.0.0.0 --port $PORT`
6. Add environment variable: `GEMINI_API_KEY=your_key`
7. **Important:** Also upload `data/shl_assessments.csv` and `embeddings/assessment_embeddings.pkl` to your repo (or build them as part of startup)

### Deploy the frontend:
1. New Web Service → same repo
2. Start command: `streamlit run app.py --server.port $PORT --server.address 0.0.0.0`
3. Add environment variable: `API_URL=https://your-api-url.onrender.com`

## 🔬 Technical Details

| Component | Choice | Reason |
|---|---|---|
| Embeddings | `text-embedding-004` (Gemini) | Free, high-quality, 768-dim |
| LLM | `gemini-1.5-flash` | Free tier, fast, smart |
| Vector Search | Cosine similarity (NumPy) | Simple, no infra needed, fast for <1000 items |
| Query Understanding | Gemini structured output | Extracts skills, types, duration constraints |
| Reranking | Gemini LLM reranker | Ensures balance of test types (K+P) |
| API | FastAPI | Industry standard, auto docs, fast |
| Frontend | Streamlit | Fast to build, easy to deploy |

## 📊 Evaluation Metric

**Mean Recall@10**: measures what fraction of the ground-truth assessments appear in the top-10 recommendations, averaged over all queries.

```
Recall@10 = (# relevant in top-10) / (# total relevant)
Mean Recall@10 = average over all queries
```

## 📝 Notes

- The scraper collects **Individual Test Solutions only** (ignores Pre-packaged Job Solutions)
- Embeddings are cached to disk after first run — subsequent startups are fast
- The LLM reranker ensures balanced recommendations when a query spans multiple skill domains
